# GeoDoc AI v0.5.3 — Readable Answer Agent Fix

## Что исправлено

- Короткие запросы по геологическим терминам вроде `Жигулевский ярус` теперь получают человеческий ответ: что это, где находится в разрезе, литология, мощность/глубина, источники.
- Для таких запросов таблица больше не подменяет ответ фразой «найдена релевантная таблица». Таблица остаётся evidence во вкладке «Таблицы».
- Добавлен backend fallback-синтез, который работает даже без Qwen API.
- Улучшен prompt для LLM: для ярусов/горизонтов/пластов нужно давать объяснение, а не техническую выдачу поиска.
- Исправлено отображение текстовых источников во frontend: PDF-строки больше не показываются рваными короткими строками, текст склеивается в нормальные абзацы.
- Исправлен рендер visual pages: `vision_max_pages` теперь трактуется как лимит количества страниц, а не как максимальный номер страницы. Поэтому рисунки на страницах 44/70 больше не пропускаются только из-за номера страницы.
- Добавлена чистка табличных ячеек от технических переносов/битых символов.

## Ограничение

Если сама таблица в PDF извлечена pdfplumber плохо, система теперь не выдаёт её повреждённые ячейки как основной ответ. Для идеальной таблицы всё равно нужен OCR/Vision или ручная проверка строки в источнике.
