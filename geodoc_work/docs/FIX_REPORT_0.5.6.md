# Fix Report v0.5.6 — Vision Quality Boost (no external APIs)

## Проблема
Агент не мог нормально читать геологические карты и рисунки:
- `deepseek-ocr` получал изображение без предобработки → мелкие подписи (номера скважин, значения изолиний) не читались
- Один запрос на всё изображение → модель "размазывала" внимание, пропускала детали
- Промпт был декларативным ("смотри внимательно") без структуры анализа
- `_local_map_reading` использовал только 5 цветов и примитивный threshold → слабая поддержка для answer agent
- DPI рендеринга PDF: 170 (мало для мелких подписей)
- `_guess_visual_type` не использовал детекцию линий

## Решения (только Pillow + prompting, без внешних API)

### 1. Предобработка изображений (`vision_agent.py`)
**`_enhance_for_ocr()`** — перед отправкой в deepseek-ocr:
- Contrast ×1.25 → улучшает читаемость тонких изолиний
- Sharpness ×1.4 → чёткость текста и маркеров скважин  
- UnsharpMask (radius=1, 80%) → восстановление края линий

### 2. Multi-crop второй проход (`vision_agent.py`)
**`_run_region_passes()`** — после основного анализа карты запускает 6 целевых кропов:
- `title_area` (верхние 12%) — читает заголовок и номер рисунка
- `legend_right` (правые 25%) — читает все условные обозначения
- `legend_bottom` (нижние 15%) — читает подписи снизу
- `center_map` (центральные 64%) — анализирует изолинии и контуры
- `left_edge` / `bottom_edge` — читает координатную сетку

Кропы масштабируются до ≥400×300px для лучшего OCR.

**`_merge_region_crops()`** — объединяет результаты кропов с основным payload:
- Добавляет в `visible_text` уникальные строки из всех регионов
- Извлекает заголовок если основной проход его не нашёл
- Парсит координаты из краевых кропов (паттерн `\d{4,}` и `°`)
- Дополняет легенду из legend-кропа

### 3. Chain-of-Thought промпт (`prompts/geology.py`)
`VISUAL_GEOLOGY_ANALYSIS_PROMPT` переписан с 9 явными шагами:
1. Классификация (map/figure/table)
2. Заголовок и подпись (дословно)
3. Легенда (каждый символ)
4. Скважины (номера, типы, позиции)
5. Контуры и изолинии (числа, ВНК/ГНК)
6. Координаты и ориентация
7. Весь видимый текст
8. Геологические наблюдения
9. Ограничения

Добавлен `VISUAL_REGION_ANALYSIS_PROMPT` для анализа конкретного региона карты.

### 4. Улучшенный `_local_map_reading()` (`document_parser.py`)
- 11 цветов вместо 5: добавлены orange, brown, purple, cyan, grey
- Пространственная сетка 4×4 для определения позиций зон
- **Edge density через `ImageFilter.FIND_EDGES`** — детекция изолиний без внешних библиотек
- **Детекция легенды** — анализ правой полосы 20% на white_ratio
- **Счётчик маркеров скважин** — tile-based подсчёт красных/тёмных кластеров 8×8px
- Описания позиций: "северная, западная часть" вместо "слева"
- Summary содержит количественные данные: % покрытия, плотность линий, ~N скважин

### 5. Улучшенный `_guess_visual_type()` (`document_parser.py`)
- Расширена палитра (orange, purple) → лучше ловит нестандартные карты
- Добавлена детекция edge density → карты с изолиниями теперь всегда map
- Три независимых критерия (fill+edge, colour, points) вместо одного

### 6. DPI рендеринга: 170 → 220 (`config.py`)
Больше пикселей на знак → deepseek-ocr читает мелкие подписи лучше.

### 7. Улучшенный `OCR_STRUCTURING_PROMPT`
Добавлены правила для: диапазонов глубин, пустых ячеек таблиц, продолжений таблиц, координат, многострочных подписей рисунков.

## Архитектура пайплайна (после правок)

```
PDF страница (220 DPI PNG)
        │
        ▼
_enhance_for_ocr()          ← contrast + sharpen + unsharp
        │
        ▼
Pass 1: deepseek-ocr        ← Chain-of-Thought (9 шагов) → primary payload
        │
        ▼ (если map или figure)
Pass 2: 6 regional crops    ← title / legend / contours / coordinates
        │
        ▼
_merge_region_crops()       ← объединение visible_text, coords, title, legend
        │
        ▼
_merge_local_map_reading()  ← обогащение edge_density, well_count, zones
        │
        ▼
DocumentArtifact (map)      → RAG answer agent
```

## Влияние на ответы
- Заголовки карт теперь читаются дословно (через title_area кроп)
- Координатная сетка попадает в artifact.text → поиск по координатам работает
- Количество скважин указывается количественно (~N маркеров)
- Легенда содержит геологический смысл каждого цвета
- Ответ "Ограничения" точно описывает что не прочитано
